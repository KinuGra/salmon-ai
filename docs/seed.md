## モックデータシード（Big Five）

#### 概要

AI機能（工数見積もりアラート・自己分析レポート）のUIをテストするため、心理学の「Big Five」に基づく3種類の行動パターンを持ったモックユーザーのデータをDBに投入するSQLシードファイルです。

---

#### 背景・目的

- 「AIの見積もりアラートが正しく表示されるか」「自己分析レポートが傾向を正確に分析できているか」をUI上で確認する
- 3人のユーザーはそれぞれ異なる行動傾向を持つため、レポート・アラート・タイムブロッキング画面の各状態を網羅的に検証できる
- モック認証（`MockAuth`）のIDを切り替えるだけで各ユーザーに即座に切り替えられる

---

#### ファイル

- `infra/seed.sql`

---

#### 実行方法

```bash
docker exec -i <postgres_container> psql -U <user> -d <dbname> < infra/seed.sql
```

冪等性あり（`ON CONFLICT (id) DO NOTHING`）。すでにデータが存在する場合は何も変更しない。

---

#### ユーザー切り替え方法

`backend/cmd/server/main.go` の `MockAuth` に渡すIDを変更する。

```go
// ユーザー1: 田中 誠一（誠実性が高い）
r.Use(middleware.MockAuth(10))

// ユーザー2: 鈴木 彩花（外向性・楽観的・過信型）
r.Use(middleware.MockAuth(20))

// ユーザー3: 佐藤 健太（慎重すぎ・コンフォートゾーン型）
r.Use(middleware.MockAuth(30))
```

---

#### 各ユーザーの特性とデータ設計

**ユーザー1 — 田中 誠一（ID: 10）誠実性が高い**

| 項目 | 内容 |
|---|---|
| 1日タスク数 | 3件（適切） |
| 達成率 | 100% |
| 自己見積もり vs AI見積もり | 完全一致 |
| レポート診断 | 最適ゾーン |

- 過去4日間のすべてのタスクが `achievement_rate: 100`
- `estimated_hours` と `ai_estimated_hours` が常に一致しており、見積もりアラートは発生しない
- レポートは「現在の計画スタイルを維持してください」というポジティブな内容

---

**ユーザー2 — 鈴木 彩花（ID: 20）外向性・楽観的・過信型**

| 項目 | 内容 |
|---|---|
| 1日タスク数 | 4〜5件（多すぎ） |
| 達成率 | 約31% |
| 自己見積もり vs AI見積もり | 自己1h / AI 3〜4h（大きな乖離） |
| レポート診断 | 過信・オーバーコミット |

- 企画・資料作成系のタスクで `estimated_hours: 1.0` に対し `ai_estimated_hours: 3.0〜4.0` が設定されており、**見積もりアラート（オレンジの「！」）が発生する**
- 多くのタスクが `achievement_rate: 0` または `30` で未達
- レポートは「楽観的すぎる見積もりにより過信している。AIの予測を参考にバッファを取るよう」という指摘内容

---

**ユーザー3 — 佐藤 健太（ID: 30）慎重すぎ・コンフォートゾーン型**

| 項目 | 内容 |
|---|---|
| 1日タスク数 | 2〜3件（少なすぎ） |
| 達成率 | 100% |
| 自己見積もり vs AI見積もり | 自己4h / AI 2h（保守的すぎ） |
| レポート診断 | コンフォートゾーン滞在 |

- 1日のタスク数が極端に少なく、すべて `achievement_rate: 100`
- `estimated_hours` が `ai_estimated_hours` の約2倍で、過剰なバッファを確保している
- レポートは「確実にこなせる範囲に留まっており成長機会を逃している可能性がある」という分析内容
