# salmon-ai — AIと振り返るタイムブロッキングアプリ

## 解決する課題

「タスクを書き出しても、1日が終わると何も終わっていない」

この体験には、いくつかの共通した原因があります。

| 課題                       | よくある状況                                     |
| -------------------------- | ------------------------------------------------ |
| **見積もりが甘い**         | 「1時間で終わる」と思ったタスクが3時間かかった   |
| **スケジュールが非現実的** | タスクを詰め込みすぎて、バッファがゼロ           |
| **振り返りをしていない**   | 失敗パターンに気づかず、毎日同じミスを繰り返す   |
| **自分の傾向を知らない**   | いつも優先度の低いタスクだけこなして満足している |

salmon-ai はこれらを **タイムブロッキング × AI** で解決します。タスクを登録するだけでAIが工数を見積もり、1日の計画の無理を指摘し、夜には振り返りをAIと対話し、蓄積されたデータから自分の行動パターンをレポートします。

---

## 機能一覧

### 1. タスクリスト

タスクの登録・管理を行う起点となる画面です。

**実装済みの機能**

- タイトル・詳細・優先度（高/中/低）・期限・カテゴリ（色ラベル）の設定
- **タスク作成時にAIが自動で工数を見積もる**（`ai_estimated_hours`）
- 自分の見積もりとAIの見積もりを並べて表示
- 見積もりの乖離が大きい場合に **オレンジの「！」アラートを表示**
- アラートを押すと乖離の詳細と再見積もりボタンが表示される
- 達成度が100%になると自動で完了に切り替わる（それ以外の達成度では完了にはならない）

```
タスク一覧
┌─────────────────────────────────┐
│ 企画書を作る          自己 1h  │
│ 優先度: 高  期限: 今日  AI  3h ！│
│                      ↑乖離アラート│
└─────────────────────────────────┘
```

---

### 2. タイムブロッキング（タイムライン）

1日のスケジュールをタイムラインに配置し、実行と記録を行う中心画面です。

**実装済みの機能**

- タスクブロックを**ドラッグ＆ドロップ**でタイムラインに配置
- 開始・終了時刻から工数を自動計算
- 各タスクブロックに**達成度（未入力 / 30% / 70% / 100%）を直接入力**
  - 未入力と入力済みで色を分けて視覚化
- 右下に自分の見積もり工数を表示（アラート状態はオレンジで強調）
- **タイムブロッキングしていないタスクをインボックスとして一覧表示**（優先度・期限でソート）
- AIスケジューリングサポートボタンで現在の配置を診断

---

### 3. スケジューリングサポート

AIが現在のタイムブロッキングを診断し、改善点を指摘します。

**実装済みの機能**

- **バッファ不足の検出**：スケジュールが詰まりすぎている場合に警告
- **優先度バイアスの検出**：優先度の低いタスクばかりこなしていないかを指摘
- 問題がない場合は「バランスが取れています」と肯定フィードバック

---

### 4. 振り返り・AIとの対話

1日の終わりに振り返りを入力し、AIとチャット形式で深掘りします。

**実装済みの機能**

- 初回利用時に**振り返りの重要性を説明するチュートリアル**を表示
- 「今日はどうでしたか？」という問いかけからチャットが始まる
- **SSEストリーミング**によるリアルタイムのAI応答表示
- 振り返りの過去ログを日付一覧から振り返れる履歴画面
- AIの返答は**Markdownレンダリング**で表示（太字・箇条書き・コードブロック対応）

---

### 5. 自己分析レポート

蓄積されたタスク・タイムブロッキング・振り返りデータをAIが分析し、Markdown形式のレポートを生成します。

**実装済みの機能**

- 「レポートを生成する」ボタンで最新データを元にAIがレポートを作成
- 生成したレポートはDBに保存され、次回以降も最新のものを表示
- レポートの分析項目：
  - **パフォーマンス要約**：達成率・完了件数の全体傾向
  - **成功・失敗パターン**：達成率が高い/低い条件の抽出
  - **工数見積もりの傾向**：楽観的すぎるか、悲観的すぎるかの分析
  - **タスク量の自己認識**：
    - 少なすぎ → コンフォートゾーンに留まっている可能性を指摘
    - 多すぎ → 過信・オーバーコミットの傾向を指摘
    - 適切 → 現在のバランスを肯定しアドバイス

---

### 6. 進捗・サマリー

週次・月次の活動を俯瞰する画面です。

**実装済みの機能**

- **GitHubスタイルの草グラフ**（過去20週分のタスク完了数を可視化）
- 週次・月次タブで切り替え
- 達成タスク数・活動日数・達成率を表示
- **前週/前月との比較**（増減バッジで差分を表示）
- AIによる一言レビュー（現在の活動傾向へのコメント）

---

## 画面構成

```
/ (ホーム)
├── /tasks              タスクリスト・工数見積もりアラート
├── /time-blocking      タイムブロッキング・スケジューリングサポート
├── /reflection         振り返り・自己分析レポート
│   ├── 自己分析レポート表示
│   ├── AIチャット（今日の振り返り）
│   └── 過去の振り返り履歴
└── /stats              進捗・サマリー・草グラフ
```
